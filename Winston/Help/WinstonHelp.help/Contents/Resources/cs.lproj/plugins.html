Pluginy

Volitelné komunitní doplňky rozšiřující Winston.

Pluginy jsou malé doplňky napsané v JavaScriptu. Mohou například doplňovat chybějící údaje o knihách nebo vytvářet přehledy. Ve výchozím stavu jsou vypnuté: nic se nespustí, dokud plugin nenainstalujete a nezapnete, a každý smí dělat jen to, co mu výslovně povolíte.

Jsou pluginy bezpečné?

  • Plugin běží v samostatném ukončitelném worker procesu bez přímého přístupu k vašim souborům, síti nebo jiným aplikacím. Může používat jen konkrétní možnosti poskytované Winstonem.
  • Před prvním spuštěním Winston přesně ukáže, co chce plugin dělat, například „Číst metadata knihovny“, a požádá vás o potvrzení.
  • Plugin nikdy nemůže přepsat metadata, která jste zadali nebo upravili; smí pouze doplnit prázdná pole.
  • Online vyhledávání respektuje přepínač Nastavení → Metadata, takže při vypnutých online metadatech plugin neprovádí žádná síťová volání.
  • Pokud plugin opakovaně chybuje, zasekne se nebo překročí limit prostředků, Winston jeho proces ukončí a umístí jej do karantény; jeho kód dál neběží.

Instalace pluginu

  • Otevřete Nastavení → Pluginy a klikněte na Otevřít složku pluginů. Nachází se v ~/Library/Application Support/Winston/Plugins/.
  • Vložte dovnitř složku pluginu. Každý plugin je jedna složka obsahující soubor manifest.json a skript.
  • V Nastavení → Pluginy klikněte na Obnovit. Plugin se objeví v seznamu jako Vypnutý.

Zapnutí a oprávnění

  • Plugin zapnete jeho přepínačem.
  • Winston zobrazí okno Zapnout „…“? se seznamem požadovaných oprávnění. Přečtěte si je a klikněte na Zapnout nebo Zrušit.
  • Běžná oprávnění jsou Číst metadata knihovny, Doplňovat chybějící metadata knih, Načítat metadata z online katalogů a Zobrazovat oznámení.
  • Souhlas s oprávněními je svázán s digestem celého obsahu pluginu. Změní-li se kterýkoli soubor, Winston požádá znovu o potvrzení i bez zvýšení verze; aktualizace tak nemůže potají zdědit přístup.

Správa pluginů

  • Každý plugin zobrazuje stav: Aktivní, Vypnutý, Neplatný (složka neobsahuje použitelný plugin), Selhal (nepodařilo se jej spustit a zobrazí se důvod) nebo V karanténě (vypnutý po opakovaných chybách).
  • Rozbalte Protokol a zobrazte poslední zprávy pluginu. Pomohou zjistit, co provedl nebo proč selhal.
  • Volba Zobrazit ve Finderu otevře složku pluginu.
  • Plugin můžete kdykoli vypnout. Plugin v karanténě lze znovu zapnout a vyzkoušet.

Řešení potíží

  • Plugin se po přidání nezobrazuje. Název jeho složky musí odpovídat hodnotě id v souboru manifest.json a složka musí obsahovat manifest i skript. Klikněte na Obnovit.
  • Zobrazuje se „Neplatný“. Manifest chybí nebo má chybný formát; důvod je uveden vedle stavu.
  • Zobrazuje se „Selhal“. Skript pluginu při spuštění narazil na chybu. Podrobnosti najdete v Protokolu.
  • Zobrazuje se „V karanténě“. Plugin příliš často chyboval nebo trval příliš dlouho. Zbytek Winstonu to neovlivní; po opravě autorem jej můžete znovu zapnout.

Vytvoření vlastního pluginu

  • Pluginy jsou běžný JavaScript, který používá malé API Winston. Může číst knihovnu, dohledávat metadata, ukládat menší množství vlastních dat a zobrazovat oznámení.
  • Příklad připravený ke zkopírování a podrobný návod pro začátečníky „Your First Metadata Fetcher“ jsou v projektu Winston ve složkách docs/example-plugin/ a docs/WritingPlugins.md; kompletní referenci API najdete v docs/PluginAPI.md.
